🔥 今日高光

  1. Claude Code 上线 /checkup：一键做「上下文卫生」。Boris Cherny 发布新命令 /checkup，一次跑完 7 件事：① 清理没用的 skills / MCP / plugins 省 context；② 把本地 CLAUDE.md 和 checked-in 版本去重；③ 把臃肿的 root CLAUDE.md 拆成嵌套 CLAUDE.md + skills；④ 关掉慢 hooks；⑤ 升级到最新版；⑥ 默认开启 auto mode；⑦ 预批常被拒的只读命令。所有改动前都会先跟你确认。这条拿下全场最高赞（❤6631 / 🔁519）——信号很明确：agent 编排的下一个战场是「上下文管理」本身。🔗 https://x.com/bcherny/status/2074997570317779038
  2. 「重写可以又好、又便宜、又快」——一次工程心智更新。Thariq（Claude Code @Anthropic）借 Bun 团队整仓重写立论：「这应该大幅更新你对软件工程的认知模型——rewrites can be good, cheap and fast。当然大多数 app 没 Bun 那么可测试、可验证，但模型会持续变强、把这些空白填上。」把「重写 = 高风险」的老直觉，正式改写成「在可验证系统里，重写是廉价操作」。🔗 https://x.com/trq212/status/2074993112217461020
  3. Grok 4.5 一周内被 Vercel / Cursor / Box 同步接入。Guillermo Rauch 宣布 Grok 4.5 面向全部 Vercel 客户开放；Ryo Lu 邀大家在 Cursor 里试用、称「新纪元的开始」；Aaron Levie（Box CEO）点评其「cost + performance 都是又一个强力入口，尤其在 legal / 专业服务 / 医疗等垂直域」。模型层竞争已进入「同一款新模型当周铺满所有 dev 平台」的节奏。🔗 https://x.com/rauchg/status/2074920996201796067 · https://x.com/ryolu_/status/2074951992884244606 · https://x.com/levie/status/2075073587015516228

🎧 Podcast

AI & I by Every — 《How a Writer Uses AI Without Losing His Voice》（2026-07-08）

一位作家分享他如何在用 AI 的同时守住自己的「声音」与深度注意力：早上不碰手机、不上网直到午后很久，因为「一碰手机就有化学层面的切换，进不了深度思考 / 深度专注」；他甚至专门用一台「什么有趣的事都干不了」的笔记本来写作。核心张力：AI 「既能帮你做出惊人的工作，也像一台老虎机」——高度 dopamine 驱动。给所有 builder 的提醒：主动设置摩擦力（barriers），保护那部分「不可被替代的自己」。 🔗 https://www.youtube.com/watch?v=7ND0lQmLJlA
