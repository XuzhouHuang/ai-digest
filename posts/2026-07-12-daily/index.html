🔥 今日高光#

  1. Agent 正在同时成为买家和卖家。Stripe 已把『买按钮』铺进各大 AI surface：与 Google 合作让商家直接在 AI Mode / Gemini 里成交，与 Microsoft、OpenAI 在 Copilot / ChatGPT 里做商品可发现，与 Meta 一起在广告里直接结账。底层是与 OpenAI 合建的 agentic commerce protocol。
  2. 自动驾驶式的『代理自主等级』。Stripe 内部用 L1–L3 给 agentic commerce 分级：L1 是人做全部决策、agent 只执行交易；L3 是 MPP 版本、agent 近乎完全自主。目前消费者侧『大多停留在 L2』——把选品部分委托给 AI，但『还没到一句话就一键订完整个暑假旅行』的世界。
  3. Vibe coding 已解决，瓶颈转移到 Vibe deployment。给出的硬数据很说明问题：Stripe 文档的 agent 流量一年涨了 10 倍以上、现已占全部文档流量约 40%；Stripe CLI 的 API 资源请求 70% 来自 agent。写代码不再是难点，把应用真正部署上线（配数据库、auth、hosting、密钥）才是 binding constraint——为此 Stripe 上线了 Stripe Projects，联合 Vercel / Supabase / Cloudflare / Twilio / Clerk 等做部署编排。
  4. Per-seat 定价在 AI 时代失效，usage-based 成刚需。SaaS 时代边际成本近零、按席位收费很美好；但 AI 每个 prompt / API call / task 都有真实边际成本，『客户怎么用直接决定你赚钱还是亏钱』。Lovable、ElevenLabs 都从纯订阅转向 subscription + usage-based 的混合计费，agent 以机器速度消费更是逼出实时计量 + 实时计费（Metronome + Tempo 组合）。
  5. Token theft：被严重低估的头号新型欺诈。『骗子发现在 AI 里根本不用偷钱或凭证，直接偷 token 就行。』三种当下主流手法：① multi-account abuse——超过六分之一的 AI 注册是刷新用户额度的滥用；② free trial abuse——半年内在 Stripe 上翻了一倍多、增量几乎全来自 AI，甚至有『免费试用卡』的灰产链条；③ 用量计费下的『吃霸王餐』——月底账单直接不付。关键在于:这在 SaaS 时代因为边际成本为零而无所谓，在 AI 时代却关乎生死。

🎙️ PODCASTS#

The MAD Podcast with Matt Turck —《Stripe’s AI Chief: How AI Agents Will Buy, Sell, and Pay》#

一句话要点：当 agent 成为真正的买家和卖家，整个商业的『经济基础设施』——从定价、计费到欺诈防护——都要重写一遍；其中最被低估、也最致命的，是 token theft。

背景与嘉宾：对话嘉宾是 Emily Sanders，Stripe 数据与 AI 负责人。因为 Stripe『几乎服务所有 AI 公司』，她有一个观察全行业欺诈与计费模式演进的『前排座位』。这一年里，去年还停留在假设层面的『agent 作为买家』，如今已经有真实部署的基础设施、真实公司在上面构建、真实可学习的模式。

核心洞察：

  • agentic commerce 已经落地成产品。不只是科幻里那种『agent 完全自主发现服务、自己下单』的极端，更普遍的是『你在 AI surface 里问跑鞋，答案越来越带一个 buy button』。Stripe 把这套能力打包成 agentic commerce suite：商家暴露商品目录/库存/价格 → 消费者授权 agent 代付 → agent 安全执行交易。早期采用者包括 JD Sports、Fanatics、Best Buy、Coach、Kate Spade，平台侧有 Wix / Shopify / BigCommerce。

  • 『vibe coding 已解决,vibe deployment 成了新长杆』。用 Stripe 自家数据佐证：文档 agent 流量一年涨 10 倍+、CLI 请求 70% 来自 agent——写代码这一环基本被 AI 填平，真正卡住创业者的是把应用部署上线。Emily 直言 Stripe 关心这件事的理由很诚实：『it was becoming the bottleneck……我们的使命是提升互联网的 GDP』，去掉部署摩擦=直接扩大互联网经济。

  • token 作为『钱』改变了 SaaS 的黄金公式。SaaS 多一个客户几乎零成本，所以定 subscription / per-seat 很爽；AI 每次调用都有真实推理成本，于是 usage-based 计量成为刚需。而 agent 会以机器速度消费，逼出实时计量 + 实时计费；甚至会计岗位都在变——她遇到一位 AI 公司的『会计+工程师混合体』,工作不只是关账，而是从 rev-rec 数据里揪出欺诈异常和产品漏洞。

  • 最狠的一段是 token theft。原话最扎心：

    “Fraudsters have figured out that in AI, you actually don’t really need to steal money or credentials. You can just steal tokens. And the scale of this actually shocked me when I looked at the data. So more than one in six signups at AI companies are this kind of abuse.”

    偷来的 token 有真实价值:可以直接转售（淘宝/暗网上以低于成本价出售登录凭证）、可以套一层 wrapper 冒充成自己的产品卖、甚至有人批量生成音乐传到 Spotify / Apple Music 刷假播放骗版税。关键转变在于:这些本质是 first-party abuse（资源转售、账号共享、多开、免费试用滥用）,过去在 SaaS 里因为边际成本为零而无关痛痒,如今在 AI 里却能让产品经济模型迅速崩掉。Stripe 的 Radar 已从『交易欺诈』扩展到『端到端全生命周期滥用』:登录时、free trial 开始时、用量累积时各有一个 API,你把已知信息发过去、拿回一个风险分再决定放行/加验证/切服务。

  • 一个反直觉的结论:半年前不少 AI 公司想靠『干脆砍掉 free trial、只做企业销售』来规避滥用;现在这种说法少多了——因为大家都意识到 agent 也会是自己的买家,而 agent 买家意味着必须保留 self-serve / PLG 通道,没人愿意为了防欺诈砍掉增长源。安全地实现,才谈得上兑现 AI 的经济上行空间。
