Claude：agent 产品开始补上两个关键层——质量闭环与长期记忆#

Claude 官方今天两条动态连起来看，方向非常清楚。Outcomes 不是简单的 prompt 优化，而是把 rubric、grader、iteration 变成标准工作流，让 agent 不只是“给答案”，而是围绕质量标准反复收敛。

Dreaming 则补上了另一个长期缺口：agent 往往每次都像失忆重来，而 Dreaming 会回看过去 session、提取模式、整理 memory，让系统具备一定的“经验沉淀”。如果 Outcomes 负责把单次任务做对，那么 Dreaming 负责让多次任务越做越像同一个成熟 teammate。

🎧 Podcast#

Anthropic 的 Boris Cherny：coding 的真正变化，不是“会不会写”，而是“软件生产会不会变成持续运行的 agent system”#

The Takeaway： Boris Cherny 给出的最重要判断是，coding 的第一阶段已经不是“让模型帮你补全几行代码”，而是把软件开发重构成一个可并行、可持续、可调度的 agent production system。

作为 Claude Code 的核心创造者，Boris 的视角特别有价值，因为他不是站在外围评论趋势，而是在直接定义下一代 coding interface。他讲得最有冲击力的一点，是自己现在大量工作都在手机上完成，常驻 5-10 个 session，白天有几百个 agents 同时运行，夜里会有几千个 agents 做更深的工作。最值得记住的一句原话是：“I feel like loops are the future at this point.”

这句话为什么重要？因为它把 coding agent 的价值，从“单次响应更聪明”推进到了“任务会自己持续跑”。Boris 展示的工作方式里，agent 不只是接一个 prompt 然后结束，而是会用 loop、sub agents、routines 去盯 PR、修 CI、聚类用户反馈、定时回报结果。这意味着软件开发的基本单位，正在从“开发者的一次操作”变成“一个持续执行的自动化回路”。

他还有两个判断同样值得记住。第一，coding 会像 literacy 一样普及，懂领域的人未来可能比纯工程师更适合写出真正有价值的软件。第二，Anthropic 内部真正领先的未必是模型，而是组织方式：大家都在围绕 agent workflow 重构流程、协作和角色分工。

今日一句#

今天最清楚的主线是：AI builder 的竞争，正在从“谁模型更强”切到“谁能把 agent 变成持续运行的系统能力”。 compute 是上游瓶颈，Outcomes 和 Dreaming 是中间层基础设施，Boris 展示的 loops / sub-agents / mobile workflow 则是下游工作方式的变化。下一轮优势，不只是会用 AI，而是会不会运营一整套 agent 产能。
