Stripe 的 AI 掌门：AI Agent 将如何买、卖与支付 The MAD Podcast with Matt Turck · 嘉宾：Emily Sense（Stripe 数据与 AI 负责人）

一句话 takeaway：agentic commerce 的真正瓶颈不是「让 agent 会买东西」，而是重建整个经济栈——身份、支付、定价、反欺诈——而 token 盗刷是其中最被忽视的定时炸弹。

Emily Sense 用经济学家的视角拆解了「agent 替你买卖」这件事背后的基础设施。最刺眼的数据来自反欺诈：「你其实不需要偷钱或偷凭证，你可以直接偷 token……超过六分之一的 AI 公司注册都是这类滥用。」 她把这形容为「吃霸王餐，只不过餐是 token」——这也是她眼中「整个 AI 领域最被低估的话题」。

在支付侧，Stripe 与 OpenAI 联手做的 ACP（Agentic Commerce Protocol） 被类比为「commerce 版的 MCP」：商家把商品目录、库存、价格一次性暴露给协议，就能被所有接入的 agent 触达，而不必向每个新 agent 逐一注册；配套的 shared payment token 用 tokenized 凭证完成交易，agent 全程拿不到真实信用卡信息。落地名单已经很硬：Best Buy、Coach、JD Sports 等品牌，Shopify、Wix、BigCommerce 等平台，OpenAI、Google、Microsoft 等 agent 侧全部就位。

定价是另一条主线。Emily 直言 per-seat 定价正在破裂——当 AI 直接替代人力工作量时，按人头收费等于自我设限；AI 公司必须转向 usage-based billing，实时计量客户真实消耗并对齐底层成本，否则「客户怎么用你的产品直接决定你赚钱还是亏钱」。终局设想更大胆：不只是「Emily 授权 agent 替她买东西」，而是「Emily 有一个 agent 在替她经营一门生意」，买进卖出、自负盈亏——这正是她希望「12 个月后能讨论的世界」。
