🔥 **今日高光**

1. **Garry Tan 把「自己的 Claude Code 全套配置」开源——gstack 单日狂飙到 11 万 ⭐**
   今天最该被 builder 记一笔的，是 YC 总裁 **Garry Tan** 开源的 **gstack**（⭐112,844，今日 +649）。它不是又一个框架，而是 *"Garry Tan's exact Claude Code setup"*——把他本人日常在用的 **23 个 opinionated 工具**直接打包，这些工具分别扮演 **CEO / Designer / Eng Manager / Release Manager / Doc Engineer / QA** 等角色。换句话说，他开源的不是代码，而是**一整套「用 agent 当一个虚拟团队」的工作方法论**。这条和下面 Matt Pocock 的 skills、以及昨天就在冲榜的 agentic-engineering 趋势完全咬合：**2026 年中，coding agent 的护城河正在从「模型多强」转向「你的 .claude 目录沉淀了多少可复用工作流」。**
   🔗 <https://github.com/garrytan/gstack>

2. **学术圈集体押注「Agent Memory Governance」——记忆的难点不再是「记得住」，而是「管得好」**
   HuggingFace Daily Papers 今天出现一个非常一致的信号：**多篇高质量论文同时把「记忆治理」当成 agent 的下一个硬骨头**，而不再是单纯的「记忆召回」。最有代表性的是 **GateMem**（《Benchmarking Memory Governance in Multi-Principal Shared-Memory Agents》）——它指出现有记忆 benchmark 几乎都假设「单用户」，但医院、职场、校园、家庭里的共享助手是**多个 principal 往同一个记忆池写入、又在不同角色/权限下查询**，所以记忆质量需要的是 *governance（治理）* 而不只是 recall。配合 **WorldLines**（长程具身家庭 agent 的记忆 benchmark）、**MemSlides**（把长期记忆与工作记忆分层的个性化 PPT agent），趋势已经很明确：**当 agent 开始长期、多人、跨任务地运行，「记忆怎么被治理」会成为比「上下文窗口多大」更关键的工程问题。**
   🔗 GateMem: <https://huggingface.co/papers/2606.18829>
   🔗 WorldLines: <https://huggingface.co/papers/2606.18847>

3. **「写 HTML，渲染成视频」——agentic 视频生产范式爆发式冲榜**
   今天 Trending 榜上最「热」（按单日增速）的两个项目都指向同一个方向：**让 agent 用它最擅长的 HTML/代码去生产视频**。**OpenMontage**（⭐11,069，今日 **+2,935**，全榜增速第一）自称 *"World's first open-source, agentic video production system"*，主打 12 条 pipeline、52 个工具、500+ agent skills；而 HeyGen 开源的 **hyperframes**（⭐29,782）口号极其直白——*"Write HTML. Render video. Built for agents."* 把这两个放一起看，一个新范式正在成型：**视频不再靠时间轴 GUI 拖拽，而是让 agent 写结构化标记 → 渲染管线出片**。对做内容/营销/教育类 AI 产品的 builder，这是一个值得立刻试水的方向。
   🔗 OpenMontage: <https://github.com/calesthio/OpenMontage>
   🔗 hyperframes: <https://github.com/heygen-com/hyperframes>

📄 **论文速递（HuggingFace Daily Papers · 2026-06-22）**

今日整体主题高度集中在 **Agent Memory / 长程任务 / 具身规划**，这是一个很值得标记的研究风向。

- **GateMem: Benchmarking Memory Governance in Multi-Principal Shared-Memory Agents**
  首个面向「多 principal 共享记忆」的 benchmark。核心洞察：现实部署里多个用户/角色往同一记忆池读写，记忆质量需要**治理（谁能写、谁能读、什么范围）**而不只是召回准确率。对做 to-B agent、团队助手的 builder 极有参考价值。
  🔗 <https://huggingface.co/papers/2606.18829>

- **WorldLines: Benchmarking and Modeling Long-Horizon Stateful Embodied Agents**
  指出现有长期记忆 benchmark 偏「语言检索/问答」，而具身 benchmark 又偏「短程任务」，两者中间是空白。WorldLines 用 project-driven 的方式评测**家庭场景下长程、有状态的具身 agent**如何记住用户routine、世界状态与历史交互。
  🔗 <https://huggingface.co/papers/2606.18847>

- **MemSlides: A Hierarchical Memory Driven Agent Framework for Personalized Slide Generation**
  把 agent 的记忆显式分成**长期记忆 vs 工作记忆**，用于个性化 PPT 生成——既要跨任务保留用户偏好，又要在多轮修改中稳住新约束、还能可靠地做局部编辑。是「分层记忆架构」一个很具体的落地样本。
  🔗 <https://huggingface.co/papers/2606.17162>

- **GeneralVLA-2: Geometry-Aware Reconstruction and Governed Memory for Robot Planning**
  面向通用 VLA（vision-language-action）机器人：用几何感知重建 + **受治理的记忆（governed memory）**来稳定物体形状与位姿，缓解单目重建「幻觉」。注意它也用了 "governed memory" 这个词——和 GateMem 形成跨领域呼应。
  🔗 <https://huggingface.co/papers/2606.17480>

- **Multi-Turn Reflective Masking Elicits Reasoning in Mask Diffusion Models**
  在掩码扩散模型里用「多轮反思式掩码」激发推理能力，是 diffusion-LM 推理方向的一篇新工作。
  🔗 <https://huggingface.co/papers/2606.16700>

> 完整 10 篇见：<https://huggingface.co/papers/date/2026-06-22>
