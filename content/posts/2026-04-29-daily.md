### Peter Yang（Roblox）— API / MCP 会成为下一批“基础设施缺口”

Peter Yang 这条点名很短，但含金量很高。他列出来的对象横跨内容平台、视频工具、金融、政务和医疗，意思很直接：这些系统正在快速遇到同一个问题——**agent 想帮用户干活，但入口不够结构化**。

过去产品团队追求的是网页体验；接下来，越来越多团队得补一层 machine-usable surface，也就是 API / MCP。否则，用户虽然有 agent，最后仍得手动进页面点来点去。

他另一条玩笑也戳中现实：如果 agents 能直接响应用户在社媒上的随机需求，那 product team 的角色会被重新定义。虽然是玩笑，但已经能看出 future backlog 会越来越受 agent mediation 影响。

### Zara Zhang / Garry Tan / Peter Steinberger — orchestration 与 browser control 正在变得更具体

Zara Zhang 提到，高阶用户会把日常交互收敛到一个主 agent，由它负责路由和调度 subagents。这个观察很重要，因为它说明多 agent 不是让用户同时盯多个窗口，而是要靠一个 **orchestrator** 吸收复杂度。

Garry Tan 今天的 GStack 更新，则把这种 orchestration 往 browser 里推进了一步：内置 Chromium、Claude Code CLI，再通过 `/pair-agent` 把真实 agent 和 browser control 接起来。这相当于把 terminal agent 延展成了 GUI agent。

Peter Steinberger 则给出更接地气的证明：遇到 GitHub rate limit 时，Codex 直接打开浏览器继续点网页完成动作。这类 workaround 其实在说明一件事——未来可靠 agent，不会被单一 interface 卡死，而会自动切换 execution surface。

## 🎧 Podcast

### Training Data — *From SEO to Agent-Led Growth: Profound’s James Cadwallader*

这期播客的主线很清楚：营销世界正从 blue-link search，转向 **probabilistic AI responses**；更关键的变化不是“搜索入口变了”，而是**走进互联网的主体从人变成了 agent**。

James Cadwallader 的几个要点尤其值得记：

- **这不是旧 SEO 的简单延长线。** 过去内容是给算法看、给人读；现在内容越来越像是给 agent 发现、给 agent 消费。营销对象从 human attention 逐渐转向 machine legibility。
- **Agent 会消费远比人类更长尾的 Internet。** 他举例说，ChatGPT 为了推荐一个 showerhead 用了 65 个网页。对 marketer 来说，这意味着 visibility 的表面积被大幅放大。
- **不同模型像不同 species。** Gemini 更偏 YouTube，ChatGPT 更偏 Reddit / LinkedIn，Claude 也在变得更依赖实时 web。所谓 “how do I rank in AI search” 其实不是单一问题，而是多模型、多来源、多语境优化问题。
- **Agent-led growth 的关键不是自动化旧流程，而是创造以前做不到的营销形态。** 比如用 agent 读取销售通话、聚类 objections、自动生成 battle cards，这不是把人工流程提速，而是在重写流程本身。
- **Dead Internet 的担忧是真实的。** 如果人类越来越少点击网页，广告和内容生产的激励链条就会松动，届时 social network 可能会变成更“人类化”的原创内容供给地。

这期最值得 builder 记住的一句话是：**未来营销不只是让 AI 帮你写文案，而是让 agent 直接参与发现、分析、分发和反馈闭环。**

## 今日观察

今天这些 builder 信号拼起来，出现了三条很清楚的主线：

- **平台层在重组。** OpenAI 与 Microsoft 的关系更新、Replit 对 vertically integrated platform 的押注，都说明 AI 竞争正在从模型层外溢到 infra 和 distribution。
- **接口层在补课。** API / MCP、browser control、主 agent + subagents，都是在给 agent execution 补真实可用的入口。
- **商业化叙事仍有断层。** builder 们已经默认 agent 会接管更多工作流，但 large enterprise 仍在问：你到底给我带来了什么确定的 productivity？

所以今天最值得记住的，不是某个单点发布，而是这个方向：
**AI 正在从“更会回答问题”走向“更像一个可编排、可执行、可部署的系统层”。**
